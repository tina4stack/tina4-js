# Module 13: Build, Packaging & Developer Experience

## npm Package

Published as `tina4js` on npm (matching the existing package name).

```bash
# Install in a project
npm install tina4js

# Or use CDN (no install)
# <script type="module" src="https://cdn.jsdelivr.net/npm/tina4js/dist/tina4.esm.js">

# CLI (global or npx)
npx tina4 create my-app
```

### Tree-Shakeable Imports

```ts
// Import everything
import { signal, html, route, router, api, pwa } from 'tina4js';

// Import only what you need (smaller bundle)
import { signal, html } from 'tina4js/core';
import { route, router } from 'tina4js/router';
import { api } from 'tina4js/api';
import { pwa } from 'tina4js/pwa';
```

If you only use `signal` and `html`, your app ships ~1KB gzip.

## Build Tool: Vite

Why Vite over Parcel (current):
- Faster dev startup (native ESM, no bundling in dev)
- Faster HMR (module-level, not page-level)
- Better tree-shaking (Rollup under the hood)
- Smaller config surface
- Massive ecosystem (plugins, community)
- Better TypeScript support out of the box

### vite.config.ts (framework build)

```ts
import { defineConfig } from 'vite';
import { resolve } from 'path';

export default defineConfig({
  build: {
    lib: {
      entry: {
        tina4: resolve(__dirname, 'src/index.ts'),
        core: resolve(__dirname, 'src/core/index.ts'),
        router: resolve(__dirname, 'src/router/index.ts'),
        api: resolve(__dirname, 'src/api/index.ts'),
        pwa: resolve(__dirname, 'src/pwa/index.ts'),
      },
      formats: ['es', 'cjs', 'iife'],
      name: 'Tina4',
    },
    rollupOptions: {
      output: {
        // Ensure clean file names
        entryFileNames: '[name].[format].js',
      },
    },
    minify: 'terser',
    terserOptions: {
      compress: { passes: 3 }, // aggressive minification
    },
  },
});
```

### vite.config.ts (user project, auto-generated by CLI)

```ts
import { defineConfig } from 'vite';

export default defineConfig({
  server: {
    port: 3000,
    // Proxy API calls to tina4-php/python in dev
    proxy: {
      '/api': {
        target: 'http://localhost:7145',
        changeOrigin: true,
      },
    },
  },
  build: {
    outDir: 'dist',
    // Generate small chunks
    rollupOptions: {
      output: {
        manualChunks: undefined, // let Vite optimize
      },
    },
  },
});
```

## Developer Experience: What It Feels Like

### Getting Started (30 seconds)

```bash
npx tina4 create my-app
cd my-app
npm run dev
```

Browser opens. You see a working app. Edit a file, see it update instantly.

### Project Created By CLI

```
my-app/
  src/
    components/
      app-root.ts         # Root component with a counter example
    routes/
      index.ts            # Home route
    pages/
      home.ts             # Home page
    public/
      css/
        default.css       # Minimal reset + utility classes
  index.html              # Entry point
  tina4.config.ts         # Framework config (router mode, API url, etc.)
  package.json            # Already has tina4js + vite
  tsconfig.json           # TypeScript config
  TINA4.md                # AI context file
  .gitignore
```

### What `npm run dev` Does

1. Starts Vite dev server on port 3000
2. Serves `index.html`
3. Native ESM — no bundling, modules loaded directly
4. HMR — edit a component, only that component refreshes
5. If `tina4.config.ts` has API proxy, forwards `/api/*` to backend
6. TypeScript compiled on the fly (no separate tsc step)
7. SCSS compiled automatically if any `.scss` files exist

### What `npm run build` Does

1. Vite builds to `dist/`
2. Tree-shakes unused framework code
3. Minifies with Terser
4. Generates source maps
5. Output: `dist/index.html` + `dist/assets/*.js` + `dist/assets/*.css`
6. Ready to deploy anywhere

### What `npm run build -- --target php` Does

1. Builds JS bundle to `src/public/js/tina4.bundle.js`
2. Converts `index.html` to `src/templates/index.twig`
3. Injects `{{ initialState | json_encode }}` for server-side state
4. Prints: "Ready! Run your tina4-php server to see it."

### Error Messages That Help

```

        HTML attributes are kebab-case. Change to: static props = { 'user-name': String }

        Return html`...` or a string from your route handler.

```

### TypeScript Autocomplete

Full IntelliSense in VS Code / WebStorm:
- `signal<T>(value)` — typed signals
- `html` — template literal syntax highlighting (via lit-html plugin)
- `Tina4Element.prop('name')` — autocomplete from `static props`
- `api.get<User[]>('/users')` — typed API responses
- `route('/path', handler)` — handler params typed from path

### Hot Module Replacement

```
Edit src/components/my-card.ts
  -> Vite detects change (< 50ms)
  -> Only my-card instances re-render
  -> State preserved (signals keep their values)
  -> No full page reload
```

## CI/CD Integration

### GitHub Actions Example

```yaml
name: Build & Test
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm test
      - run: npm run build
      - run: npm run test:size   # Fail if over 3KB gzip budget
```

### Size Budget Enforcement

The `test:size` script runs `tests/size.test.ts` which:
1. Reads the built bundle
2. Gzips it
3. Asserts < 3KB
4. Prints actual size

This runs in CI on every PR — no accidental bloat.
